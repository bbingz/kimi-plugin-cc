---
description: Resume a specific Kimi session by UUID in the current cwd
argument-hint: '<sessionId> <prompt>'
allowed-tools: Bash(node:*)
---

Run with the companion-caller env so the companion knows this is a slash-command invocation:

```bash
KIMI_COMPANION_CALLER=claude node "${CLAUDE_PLUGIN_ROOT}/scripts/kimi-companion.mjs" resume "$ARGUMENTS"
```

The companion runs in text mode and produces:
- Line 1..N: Kimi's response verbatim (may be multi-line)
- Last line: a short footer like `(session: <uuid> · model: <name> [· thinkBlocks: N])` — generated by the companion, not Claude

**If the companion exits 0**:
1. Present the full stdout **verbatim** (response + footer line). Preserve Chinese or other-language output — do NOT translate unless the user explicitly asked.
2. **MUST NOT** prepend or append commentary of your own. **No** introductory phrases like "Here is Kimi's response:". **No** closing summary.
3. After the footer line, you MAY — only if substantive and concise — add one line flagging a disagreement: "Note: Claude disagrees on X because Y." This is the ONLY allowed addition.
4. Do NOT auto-apply suggestions.

**If the companion exits non-zero**:
1. Present the stderr `Error: ...` message clearly.
2. Add **exactly one declarative suggestion** based on the error text (same mapping as `/kimi:continue`):
   - "session <sessionId> not found for this directory" → "Verify the sessionId or omit the flag to start fresh via `/kimi:ask`."
   - "session <sessionId> has no stored messages" → "The session directory exists but is empty on disk. Start a fresh session via `/kimi:ask`."
   - "invalid sessionId format" → "Pass a valid UUID (8-4-4-4-12 hex). List your sessions by inspecting `~/.kimi/sessions/` if needed."
   - "filesystem access failed" → "Check ~/.kimi/ directory and file permissions; ensure your current shell user owns the path."
3. If stderr contains a "Warning: requested --resume ... did not match returned session ..." line, surface it prominently — the session continuity contract failed even though kimi produced an answer.
4. Do NOT retry automatically.

### Notes

- `<sessionId>` (UUID, 8-4-4-4-12 hex) and `<prompt>` are both required.
- Scoped to the current cwd: a sessionId from a different cwd will fail with `session not found`, intentionally (prevents silent session forking; see `doc/probe/probe-results-v4.json` Q4.3).
- Resume validation happens wrapper-side BEFORE invoking kimi-cli (prevents the kimi-cli ghost-session bug; see `doc/probe/probe-results-v4.json` Q4.0).
