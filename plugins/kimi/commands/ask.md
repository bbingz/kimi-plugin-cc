description: Delegate a task or ask a question to Kimi
argument-hint: '[--model <model>] <prompt>'
allowed-tools: Bash(node:*)
---

Run with the companion-caller env so the companion knows this is a slash-command invocation (gates --stream to direct CLI debug only):

```bash
KIMI_COMPANION_CALLER=claude node "${CLAUDE_PLUGIN_ROOT}/scripts/kimi-companion.mjs" ask "$ARGUMENTS"
```

The companion runs in text mode and produces:
- Line 1..N: Kimi's response verbatim (may be multi-line)
- Last line: a short footer like `(session: <uuid> · model: <name> [· thinkBlocks: N])` — generated by the companion, not Claude

**If the companion exits 0**:
1. Present the full stdout **verbatim** (response + footer line). Preserve Chinese or other-language output — do NOT translate unless the user explicitly asked.
2. **MUST NOT** prepend or append commentary of your own. **No** introductory phrases like "这是 Kimi 的回答：" / "Here is Kimi's response:" / "Kimi says:". **No** closing summary. The companion's stdout is the complete user-facing payload.
3. After the footer line, you MAY — only if substantive and concise — add one line flagging a disagreement: "Note: Claude disagrees on X because Y." This is the ONLY allowed addition. Keep it to one sentence; omit entirely when you agree.
4. Do NOT auto-apply suggestions. If the user wants to act on something, they'll ask.

**If the companion exits non-zero** (check exit code / stderr):
1. Present the stderr `Error: …` message clearly.
2. If stderr contains a "Partial response" block, include it under that heading.
3. Add **exactly one declarative suggestion** based on the error keyword. Follow these literal templates — do NOT paraphrase, do NOT turn into a question. **MUST NOT end with '?'.** Declarative sentences only:
   - "not configured" → "Run `/kimi:setup` to see available model names; retry with `--model <name>`."
   - "timed out" → "Split the prompt into smaller pieces or lower the scope, then retry."
   - "interrupted" (SIGTERM/SIGINT) → "The request was interrupted. Retry when ready."
4. Do NOT retry automatically. Do NOT pose a question. The user decides whether to retry.

### Options

- `--model <name>` — pick a specific model from `configured_models` (see `/kimi:setup`)
- `--stream` (advanced) — not used from the Claude Code command; developer-only when invoking the companion directly

### Resuming a previous session

`/kimi:ask` always starts a new conversation. To resume:
- `/kimi:continue <prompt>` — most recent session for the current cwd
- `/kimi:resume <sessionId> <prompt>` — specific session by UUID

(v0.2 P2 BREAKING: `/kimi:ask --resume <sessionId>` was removed. Use the new commands above.)
